In 2026, website discoverability has two dimensions:

1. **Traditional SEO** — Google, Bing, Yandex ranking and indexing
2. **GEO (Generative Engine Optimization)** — Being cited by ChatGPT, Claude, Perplexity, Google AI Overviews, and other AI systems

## Part 2: llms.txt — The New Standard for AI Discoverability

### 2.1 What is llms.txt?

llms.txt is the "sitemap for AI." While robots.txt tells bots where NOT to go, llms.txt tells them where the valuable content IS. It's a Markdown file at the root of your website that provides structured information specifically for Large Language Models.

**Specification:** https://llmstxt.org/

**File:** `public/llms.txt`

### 2.3 llms-full.txt Implementation

**File:** `public/llms-full.txt`

This file contains the COMPLETE content of all important pages in a single Markdown document. AI systems can ingest the entire site in one request.

This should be auto-generated at build time by a script that:
1. Reads all MDX content from collections
2. Concatenates service descriptions, about info, team bios, blog posts
3. Outputs a single comprehensive Markdown file

**Build script needed:** A custom Astro endpoint or build-time script.

### 2.4 Markdown Versions of Pages

Per the llms.txt spec, provide `.md` versions of HTML pages:
- `https://sekoya.tech/about/index.html.md`
- `https://sekoya.tech/services/index.html.md`
- etc.

**Implementation:** Create Astro endpoints that generate Markdown versions.

## Part 4: GEO (Generative Engine Optimization) — Content Strategy

### 4.1 Citation-Optimized Content Structure

AI systems prefer content that is:

1. **Answer-first** — The first 200 words should directly answer the primary query
2. **Data-rich** — Include statistics, numbers, specific data points
3. **Source-attributed** — Cite sources, reference studies
4. **Regularly updated** — Add "Last updated: YYYY-MM-DD" timestamps
5. **Unique/proprietary** — Original research, case studies, proprietary insights

### 4.2 Content Types That Get AI Citations

| Content Type | Citation Likelihood | Priority |
|---|---|---|
| **Comparison/list articles** | Very High | Must have |
| **"What is X?" definitions** | Very High | Must have |
| **How-to guides with steps** | High | Should have |
| **Original research/data** | Very High | Aspirational |
| **FAQ content** | High | Must have |
| **Case studies with numbers** | High | Should have |
| **Industry benchmarks** | Very High | Aspirational |

### 4.3 Recommended Blog Content Strategy

Write blog posts specifically targeting queries that AI systems answer:

1. "What is custom software development and when do you need it?"
2. "IoT for manufacturing: complete guide [2026]"
3. "Mobile app development cost breakdown"
4. "Flutter vs React Native: which one to choose in 2026?"
5. "How to choose a software development partner"
6. "AI implementation roadmap for businesses"
7. "DevOps best practices for startups"
8. "Digital transformation checklist for SMBs"

Each post should:
- Start with a clear, definitive answer
- Include numbered lists or comparison tables
- Cite specific data points and statistics
- Have clear H2/H3 heading hierarchy
- Include an FAQ section at the bottom (with FAQPage schema)
- Show "Last Updated" date prominently

### 4.4 Entity Optimization

AI systems organize knowledge as entities. Sekoya needs to be recognized as an entity:

1. **Google Knowledge Panel** — Claim and verify via Google Business Profile
2. **Wikipedia** — Consider a stub article if Sekoya has notable public presence (careful: notability criteria)
3. **Wikidata** — Create a Wikidata entry for Sekoya (lower notability bar)
4. **Crunchbase** — Create/update company profile
5. **LinkedIn Company Page** — Keep updated with services, employee count, posts
6. **GitHub Organization** — Active repos signal technical credibility

## Part 6: AI Agent Accessibility

### 6.1 Clean, Semantic HTML

Already done:
- [x] Semantic elements (header, main, nav, footer, article, section)
- [x] Proper heading hierarchy (H1 → H2 → H3)
- [x] ARIA labels
- [x] Skip navigation

### 6.2 Content Accessibility for AI Parsing

AI agents parse HTML differently than traditional crawlers. Key principles:

1. **One topic per page** — Each page should have a clear, single focus
2. **Table data** — Use `<table>` for comparisons (AI systems extract tables efficiently)
3. **Definitive statements** — "Sekoya specializes in IoT" not "We might be able to help with IoT"
4. **Contact info in structured data** — Not just visual, but machine-readable
